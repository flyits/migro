CLI 命令参考

Migro 提供了一组简洁的命令行工具来管理数据库迁移。

命令列表
  • migro init
  • migro create
  • migro up
  • migro down
  • migro status
  • migro reset
  • migro refresh

migro init

初始化迁移环境，创建配置文件和迁移目录。

用法

bash 
migro init [flags]

参数

参数 说明 默认值
--driver 数据库驱动 (mysql/postgres/sqlite) mysql

示例

bash 
# 使用 MySQL 初始化
migro init --driver mysql

# 使用 PostgreSQL 初始化
migro init --driver postgres

# 使用 SQLite 初始化
migro init --driver sqlite

migro create

创建新的迁移文件。

用法

bash 
migro create <name> [flags]

参数

参数 说明 默认值
--table 指定表名（用于生成模板） -

示例

bash 
# 创建表迁移
migro create create_users_table

# 创建修改表迁移（指定表名）
migro create add_phone_to_users --table users

migro up

执行待执行的迁移。

用法

bash 
migro up [flags]

参数

参数 说明 默认值
--step 执行指定数量的迁移 0 (全部)
--dry-run 预览 SQL 而不执行 false
--force 跳过确认提示 false

示例

bash 
# 执行所有待执行迁移
migro up

# 只执行一个迁移
migro up --step=1

# 预览 SQL（不执行）
migro up --dry-run

# 强制执行（跳过确认）
migro up --force

migro down

回滚迁移。

用法

bash 
migro down [flags]

参数

参数 说明 默认值
--step 回滚指定数量的迁移 0 (最后一批)
--force 跳过确认提示 false

示例

bash 
# 回滚最后一批迁移
migro down

# 回滚最近 3 个迁移
migro down --step=3

# 强制回滚（跳过确认）
migro down --force

migro status

显示迁移状态。

用法

bash 
migro status

输出示例

bash 
+----+----------------------------------------+-------+---------------------+
| #  | Migration                              | Batch | Executed At         |
+----+----------------------------------------+-------+---------------------+
| 1  | 20260202150405_create_users_table      | 1     | 2026-02-02 15:04:05 |
| 2  | 20260202150410_create_posts_table      | 1     | 2026-02-02 15:04:10 |
| 3  | 20260202150415_add_phone_to_users      | -     | Pending             |
+----+----------------------------------------+-------+---------------------+

migro reset

回滚所有迁移。

用法

bash 
migro reset [flags]

参数

参数 说明 默认值
--force 跳过确认提示 false
⚠️ 危险操作

此命令会回滚所有迁移，请谨慎使用。建议在生产环境使用 --force 前先确认。

migro refresh

回滚所有迁移并重新执行。

用法

bash 
migro refresh [flags]

参数

参数 说明 默认值
--force 跳过确认提示 false
⚠️ 注意

此命令会删除所有数据并重建表结构，通常用于开发环境。
