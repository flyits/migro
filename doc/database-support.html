数据库支持

Migro 支持 MySQL、PostgreSQL 和 SQLite，并自动处理不同数据库之间的差异。

事务 DDL

数据库支持事务 DDL说明
MySQL❌ 否DDL 会隐式提交事务
PostgreSQL✅ 是完整事务支持
SQLite✅ 是完整事务支持

Migro 会自动检测数据库类型，对支持事务 DDL 的数据库使用事务包裹迁移。

类型映射

不同数据库的类型映射有所不同，Migro 会自动处理：

Migro 类型MySQLPostgreSQLSQLite
BooleanTINYINT(1)BOOLEANINTEGER
JSONJSONJSONBTEXT
自增主键AUTO_INCREMENTSERIAL/BIGSERIALAUTOINCREMENT
UUIDCHAR(36)UUIDTEXT

ALTER TABLE 限制

SQLite 对 ALTER TABLE 支持有限：

  • 支持：ADD COLUMN、RENAME COLUMN
  • 不支持：DROP COLUMN、MODIFY COLUMN（需要重建表）
⚠️ SQLite 注意事项

对于复杂的 SQLite 表修改，建议使用 Raw() 方法手动处理表重建。

MySQL 专用特性

  • 支持 UNSIGNED 整数类型
  • 支持 ENGINE、CHARSET、COLLATION 设置
  • 支持 FULLTEXT 全文索引
  • 支持 AFTER 列位置指定

PostgreSQL 专用特性

  • 原生 UUID 类型支持
  • 原生 JSONB 类型（比 JSON 更高效）
  • 支持 SERIAL/BIGSERIAL 自增类型
  • 支持 CONCURRENTLY 创建索引

外部连接支持

Migro 支持使用已有的数据库连接，而非通过配置创建新连接。这在以下场景非常有用：

  • 复用现有连接池
  • 与 GORM 等 ORM 共享连接
  • 在同一事务中执行迁移和业务逻辑
  • 测试场景中使用 mock 或内存数据库

使用 *sql.DB 连接

通过 ConnectWithDB 方法传入已有的数据库连接：

go 
import (
    "database/sql"
    "github.com/flyits/migro/pkg/driver/mysql"
    "github.com/flyits/migro/pkg/migrator"
)

// 用户已有的数据库连接
existingDB, _ := sql.Open("mysql", dsn)
defer existingDB.Close()

// 创建驱动并传入连接
drv := mysql.NewDriver()
drv.ConnectWithDB(existingDB)
defer drv.Close() // 不会关闭 existingDB

// 创建 Migrator
m := migrator.NewMigrator(drv, "./migrations", "migrations")
m.Up(ctx, 0)

使用 GORM 连接

通过 ConnectDriver 辅助函数传入 GORM 实例：

go 
import (
    migrogorm "github.com/flyits/migro/pkg/driver/gorm"
    "github.com/flyits/migro/pkg/driver/mysql"
    "github.com/flyits/migro/pkg/migrator"
    gormmysql "gorm.io/driver/mysql"
    "gorm.io/gorm"
)

// 用户已有的 GORM 实例
gormDB, _ := gorm.Open(gormmysql.Open(dsn), &gorm.Config{})

// 创建驱动并通过 GORM 连接
drv := mysql.NewDriver()
migrogorm.ConnectDriver(drv, gormDB)
defer drv.Close() // 不会关闭 GORM 连接

// 创建 Migrator
m := migrator.NewMigrator(drv, "./migrations", "migrations")
m.Up(ctx, 0)

连接所有权

Migro 使用连接所有权机制来管理连接生命周期：

连接方式所有权Close() 行为
Connect(config)驱动拥有关闭连接
ConnectWithDB(db)调用方拥有不关闭连接
ConnectDriver(drv, gormDB)调用方拥有不关闭连接
💡 提示

使用外部连接时，调用方负责管理连接的生命周期。drv.Close() 不会关闭外部传入的连接。
